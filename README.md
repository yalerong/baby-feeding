# 宝宝喂养记录 - 微信小程序

一个基于微信云开发的宝宝喂养记录小程序，支持家庭成员共享数据、历史数据导入、多维度统计图表、生长曲线追踪、疫苗接种管理。

---

## 功能特点

- ✅ **快速记录**：每次喂奶 10 秒录入，支持单条/批量添加
- ✅ **自动计算**：总奶量、母乳占比自动统计；喂养次数只统计奶量大于 0 的记录
- ✅ **家庭共享**：固定家庭码 `FAMILY`，家人自动同步，无需手动输入
- ✅ **历史编辑**：首页支持切换任意日期，查看并修改历史记录
- ✅ **统计图表**：近 7 天 / 近 30 天堆积柱状图（母乳绿 + 奶粉蓝），支持触摸滑动选中日期，选中柱子黄色高亮
- ✅ **月度汇总**：「全部」模式按月份汇总，点击月份查看每日明细；自动过滤空月份/空日期
- ✅ **大便记录**：勾选大便后，性状/颜色/量下拉选择，规范记录
- ✅ **历史导入**：支持 JSON 文件导入历史汇总数据
- ✅ **生长曲线**：独立页面记录身高体重，Canvas 绘制趋势折线图；支持**只填体重或只填身高**（如家里只有体重秤）
- ✅ **疫苗管理**：内置国家免疫规划 + 常见二类疫苗计划，自动推算接种时间，支持实际接种后联动更新后续剂次；**长按可删除**不准备打的疫苗
- ✅ **零服务器成本**：基于微信云开发，个人免费额度够用

## 数据库集合说明

### feeding_records（喂养记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 系统自动生成 |
| familyCode | string | 家庭共享码，固定为 `FAMILY` |
| date | string | 日期（YYYY-MM-DD） |
| time | string | 时间（HH:MM） |
| breastMilk | number | 母乳量（ml） |
| formula | number | 奶粉量（ml） |
| total | number | 总奶量（自动计算） |
| stool | boolean | 是否大便 |
| stoolDesc | string | 大便描述 |
| createTime | Date | 创建时间 |

> 统计口径：`feeding_records` 可以保存“只有大便、没有奶量”的记录。首页和统计页里的“喂养次数”只统计 `breastMilk > 0` 或 `formula > 0` 的记录；大便次数只按 `stool = true` 单独统计，不计入喂养次数。统计页会基于每日原始记录重新计算口径，避免旧聚合字段把纯大便记录计入喂养次数。

### growth_records（生长记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| familyCode | string | 家庭共享码 |
| date | string | 日期（YYYY-MM-DD） |
| weight | number | 体重（kg），可选 |
| height | number | 身高（cm），可选 |
| note | string | 备注 |
| createTime | Date | 创建时间 |

> 支持只记录体重、只记录身高、或两者同时记录。图表会自动过滤空值绘制。

### vaccine_records（疫苗接种记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| familyCode | string | 家庭共享码 |
| vaccineName | string | 疫苗名称 |
| category | string | 分类：一类 / 二类 / 自定义 |
| dose | number | 剂次 |
| plannedDate | string | 预计接种日期（YYYY-MM-DD） |
| actualDate | string | 实际接种日期（YYYY-MM-DD） |
| status | string | planned / completed |
| isCustomPlanned | boolean | 是否手动修正过预计时间 |
| note | string | 备注 |
| createTime | Date | 创建时间 |

> 首次设置出生日期后，系统会自动生成标准疫苗计划。不准备打的疫苗可以长按删除，也可以手动添加自定义疫苗。

## 从零开始部署教程（预计 20 分钟）

### 第一步：注册微信小程序账号

1. 访问 [微信公众平台](https://mp.weixin.qq.com/)
2. 点击右上角「立即注册」→ 选择「小程序」
3. 填写邮箱、密码完成注册
4. **重要**：在「设置」→「基本设置」里记下你的 **AppID**
5. 在「设置」→「基本设置」→「服务类目」里添加类目：
   - 选择「工具」→「记账」或「生活」→「母婴」

### 第二步：下载微信开发者工具

1. 访问 [微信开发者工具下载页](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
2. 下载 Windows 版本并安装
3. 用你注册小程序的微信扫码登录

### 第三步：导入本项目

1. 打开微信开发者工具
2. 点击「导入项目」
3. 项目目录选择本项目文件夹
4. AppID 填入你在第一步获取的 AppID
5. 后端服务选择「小程序·云开发」
6. 点击「确定」导入

### 第四步：开通云开发环境

1. 在开发者工具顶部菜单，点击「云开发」按钮
2. 按提示开通云开发（需要微信实名认证）
3. 开通后，在弹出的云开发控制台里，找到「设置」→「环境设置」
4. 复制你的 **环境 ID**（格式如 `baby-feeding-xxx`）
5. 打开 `app.js`，把 `env: ''` 改成你的环境 ID：
   ```javascript
   wx.cloud.init({
     env: '你的环境ID',
     traceUser: true
   })
   ```

### 第五步：创建数据库集合

在云开发控制台里，点击「数据库」，依次创建以下 3 个集合：

| 集合名称 | 用途 |
|---------|------|
| `feeding_records` | 喂养记录 |
| `growth_records` | 身高体重记录 |
| `vaccine_records` | 疫苗接种记录 |

> 每个集合创建后，点击「权限设置」→ 选择「所有用户可读，所有用户可写」。因为我们是通过 `familyCode` 做业务层隔离的，数据库权限可以放宽。

### 第六步：部署云函数

1. 在开发者工具左侧目录树里，展开 `cloudfunctions`
2. 对每个云函数文件夹，右键 → 选择「创建并部署：云端安装依赖」
3. 需要部署的云函数列表：
   - `addRecord`
   - `getRecords`
   - `getStats`
   - `updateRecord`
   - `deleteRecord`
   - `fixFamilyCode`（数据修复工具）
   - `fixDates`（数据修复工具）

> ⚠️ **如果右键没有「创建并部署」选项**，先右键点击 →「新建 Node.js 云函数」，然后再部署。

### 第七步：编译预览

1. 在开发者工具顶部，点击「编译」按钮
2. 右侧模拟器里应该能看到首页
3. 点击「添加喂养记录」测试录入一条数据
4. 回到首页，应该能看到汇总统计
5. 切换到「生长」和「疫苗」Tab，测试对应功能

### 第八步：导入历史数据（可选）

如果你有历史喂养数据想导入：

1. 准备 `history_import.json` 文件（每行一个 JSON 对象）
2. 打开云开发控制台 →「数据库」→ `feeding_records`
3. 点击「导入」，选择 `history_import.json`
4. 导入模式选择「插入」

> 导入前请确保 JSON 文件里的 `familyCode` 为 `FAMILY`，与代码保持一致。

## 各模块使用指南

### 喂养记录
- 首页查看今日汇总，点击「添加喂养记录」录入
- 点击任意历史记录可修改或删除
- 「批量补录历史数据」可一次性补录某天多条记录
- 可以单独记录大便；如果母乳和奶粉都为 0，这条记录会显示在记录列表里，但不会计入喂养次数

### 统计页
- 「近7天」「近30天」显示每日母乳/奶粉堆积柱状图
- 手指按住柱状图横向滑动，可实时选择日期；当前选中的柱子会显示为黄色
- 选中某一天后，上方汇总和下方每日明细都会同步切换到该日期
- 「全部」模式先展示月份列表，点击月份进入该月每日图表
- 月份列表和月明细会过滤喂养 0 次、大便 0 次、奶量 0 ml 的空日期；空月份不会显示
- 月明细横坐标会自动间隔 5-6 天显示日期，避免日期文字过密

### 生长曲线
- 支持只填体重（如家里只有体重秤）、只填身高、或两者都填
- 图表会自动过滤空值，只绘制有数据的点
- 顶部绿色卡片显示最新数据及相比上一次的变化

### 疫苗管理
- 首次进入需设置宝宝出生日期，系统自动生成接种计划
- 已接种的填上实际日期，后续剂次预计时间会自动重新推算
- 不准备打的疫苗**长按该条目**可删除；自定义疫苗在编辑页也可删除
- 可以手动修改预计日期，修改后系统会保护你的设置不再自动覆盖；点击「恢复系统预计日期」可还原

## 数据修复工具

### 修正 familyCode

如果历史导入的数据 familyCode 不一致，导致查不到记录：

```javascript
wx.cloud.callFunction({ name: 'fixFamilyCode' }).then(res => console.log(res.result))
```

### 修正记录年份

如果需要批量把数据库里某年的记录改成另一年（如 2025 → 2026）：

```javascript
wx.cloud.callFunction({
  name: 'fixDates',
  data: { fromYear: '2025', toYear: '2026' }
}).then(res => console.log(res.result))
```

## 常见问题

**Q：云开发收费吗？**
A：微信云开发有免费额度：数据库 2GB 存储、5GB 下载流量、云函数 400 万次调用/月。个人家庭用几年都不会超。

**Q：数据安全吗？**
A：数据存在腾讯云上，只有使用同一家庭码的人才能看到记录。

**Q：更新代码/重新发布体验版，以前的数据会丢吗？**
A：**不会。** 数据存在云数据库中，上传前端代码、新增页面、修改样式等操作均不会影响已有数据。只要不手动删除数据库集合或切换云开发环境，数据一直都在。

**Q：历史记录为什么在近7天/近30天里看不到？**
A：「近7天」和「近30天」是按**今天**往前推算的。如果历史数据日期较早（如上个月的），只会出现在「全部」统计里。如需让旧数据进入近7天范围，可用 `fixDates` 云函数调整年份/日期。

**Q：能不能设置喂奶提醒？**
A：当前版本为了简化没有加提醒。如需实现，可在 `pages/index/index.js` 里调用 `wx.requestSubscribeMessage`。

**Q：为什么统计页的"次数"和历史对不上？**
A：当前“喂养次数”只统计母乳或奶粉量大于 0 的记录，即 `breastMilk > 0` 或 `formula > 0`。只有大便、奶量为 0 的记录不会计入喂养次数，但仍会计入大便次数。如果导入的是汇总数据（每天1条），次数会显示为导入记录数量而非实际喂养次数。建议用「批量补录」功能补录明细。

**Q：为什么「全部」里某个月份不显示了？**
A：月份列表会过滤空数据。如果某个月所有日期都是喂养 0 次、大便 0 次、奶量 0 ml，这个月不会显示；只要有喂养记录或大便记录，月份仍会显示。

**Q：家人看不到数据？**
A：检查家庭码是否完全一致（区分大小写），且都保存到了本地。

**Q：云函数部署失败？**
A：检查 `project.config.json` 中的 `cloudfunctionRoot` 配置，确保 AppID 是真实的小程序ID而非 touristappid。

**Q：疫苗页面的预计日期是怎么算的？**
A：第1剂按「出生日期 + 标准月龄」计算；第2剂及以后按「上一剂实际接种日期 + 标准间隔天数」自动推算。你也可以手动修改预计日期，修改后系统会自动保护你的设置，不再覆盖。

**Q：疫苗计划里有些疫苗我不准备打，怎么去掉？**
A：在疫苗列表页**长按**该疫苗条目，选择「删除」即可。如果后续又想打了，可以通过「添加自定义疫苗」补回来。

**Q：生长曲线里可以只记体重不记身高吗？**
A：可以。编辑页里体重和身高**至少填一个**就能保存。如果只填体重，身高图表会自动跳过该记录，不显示空点。
